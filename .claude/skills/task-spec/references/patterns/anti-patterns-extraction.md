# Pattern: Anti-Patterns Extraction

> **Purpose**: How to mine Zone 3 anti-patterns from MCP research.
> **MCP Validated**: 2026-05-19

## The principle

Zone 3 anti-patterns aren't invented — they're EXTRACTED from real-world failure
modes found during MCP research (Phase 2 of the workflow).

Generic "be careful" is not an anti-pattern. SPECIFIC don'ts grounded in observed
failures are.

## The extraction workflow

### Step 1 — Query Context7 for the domain

```text
mcp__context7__query-docs({
  libraryId: "<library>",
  query: "common pitfalls when <task verb>"
})
```

Look for sections labeled "Gotchas", "Common Mistakes", "Anti-patterns", "Pitfalls".

### Step 2 — Query Exa for production stories

```text
mcp__exa__web_search_exa({
  query: "<library> production bug postmortem 2026",
  numResults: 5
})
```

Production incident writeups are gold — they describe real failures.

### Step 3 — Distill into specific don'ts

Convert findings into Zone 3 entries:

```markdown
## Anti-Patterns

- **Don't [specific action]** — [reason]. [What to do instead].
```

Three required components per entry:
1. **Specific action** (not generic advice)
2. **Reason** (why this fails)
3. **Alternative** (what to do instead)

## Example — extracting from a Langfuse OTEL task

### Research findings (synthesized from Context7 + Exa)

| Source | Finding |
|--------|---------|
| Context7 (Langfuse docs) | OTEL endpoint changed from `/api/public/ingestion` to `/api/public/otel/v1/traces` in v3 |
| Context7 (OpenTelemetry) | Always set `OTEL_SERVICE_NAME` or traces are orphaned |
| Exa (postmortem 2026-02) | "Forgot resource attributes; couldn't filter traces by session.id" |
| Exa (forum post 2026-03) | "Used HTTPS without cert verification disabled; silently dropped" |

### Distilled Zone 3 anti-patterns

```markdown
## Anti-Patterns

- **Don't use the v2 endpoint path** (`/api/public/ingestion`) — Langfuse v3 changed it to `/api/public/otel/v1/traces`. Use the new path.

- **Don't ship without setting OTEL_SERVICE_NAME** — orphaned traces are invisible in the UI. Always set service.name as a resource attribute.

- **Don't forget session.id and task.id resource attributes** — without them, filtering and dashboards break. They MUST be in the exporter config.

- **Don't enable HTTPS for localhost Langfuse** — self-signed certs silently drop traces. Use HTTP for local dev.
```

Each anti-pattern: specific + reason + alternative. Each grounded in a real failure mode.

## Quality bar

Good Zone 3 entries:

✅ "Don't commit `.env` to git" → specific
✅ "Don't use float for monetary fields — IEEE-754 rounding errors compound at scale; use Decimal" → reason + alternative
✅ "Don't edit `functions/adf/**` — it's auto-generated by build-copy-src.sh; modify src/core/ instead" → specific + reason + alternative

Bad Zone 3 entries:

❌ "Be careful with the database" → vague
❌ "Don't break things" → meaningless
❌ "Follow best practices" → no specifics

## Minimum count

Aim for 3-5 anti-patterns per Task-Spec. Below 3 suggests insufficient research.
Above 7 suggests the task is too broad (decompose).
